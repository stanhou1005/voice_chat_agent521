# 语音聊天机器人 — 项目架构与语音对话全流程

## 一、技术栈总览

| 层次 | 选型 | 用途 |
|------|------|------|
| 前端框架 | React 18 + Vite | SPA 页面，深色主题，无 UI 库 |
| 后端框架 | FastAPI + Uvicorn | WebSocket + REST API |
| ORM | Tortoise-ORM 1.1.7 | `settings` 表 + `session_meta` 表 |
| 智能体框架 | LangGraph 1.2 | Plan-Execute-Replan 循环架构 |
| LLM（规划/评估） | DeepSeek V4 Pro（`ChatDeepSeek`） | Planner / Replanner 节点 |
| LLM（工具执行） | DeepSeek V4 Flash（`ChatDeepSeek`） | Executor 节点的 ReAct Agent |
| Checkpointer | `AsyncPostgresSaver` | 对话历史持久化（短期记忆） |
| 会话存储 | Tortoise `SessionMeta` 模型 | 历史会话列表（长期记忆） |
| ASR | 硅基流动 SenseVoiceSmall（免费） | 语音识别 `POST /v1/audio/transcriptions` |
| TTS | 硅基流动 CosyVoice2-0.5B | 语音合成 `POST /v1/audio/speech` |
| 搜索工具 | Tavily Search（`langchain_tavily`） | Web 搜索，advanced 深度 |
| 数据库 | PostgreSQL 16 | 所有持久化存储 |
| Python | 3.12 | |
| 部署 | 本地开发 → Docker 云部署 | |

## 二、完整语音对话流程

用户从开口说话到听到回答，经历以下 7 个阶段：

```
┌──────────────────────────────────────────────────────────────────────┐
│  阶段 1：麦克风采集（前端 useRecorder.js）                            │
│                                                                      │
│  用户点击 🎤 或按空格键 → MediaStream API 获取麦克风                  │
│  → AudioContext + ScriptProcessor 持续采集 16kHz 单声道 PCM          │
│  → 实时 RMS 能量检测（> -45 dBFS = 说话中）                          │
│  → 3 秒静音自动结束 / 用户点 ⏹ 手动结束                             │
│  → 封装为 WAV → Base64 编码                                         │
└──────────────────────────────────────────────────────────────────────┘
                                    │
                                    ▼
┌──────────────────────────────────────────────────────────────────────┐
│  阶段 2：WebSocket 传输（前端 → 后端）                                │
│                                                                      │
│  客户端发送:                                                         │
│  {"type":"audio", "data":"<base64_wav>", "timestamp":1700000000}     │
│                                                                      │
│  后端 ws.py 接收 → Base64 解码 → audio_bytes (WAV)                   │
│  状态反馈: {"type":"status", "phase":"asr", "text":"正在识别语音…"}    │
└──────────────────────────────────────────────────────────────────────┘
                                    │
                                    ▼
┌──────────────────────────────────────────────────────────────────────┐
│  阶段 3：语音识别 ASR（后端 services/asr.py）                         │
│                                                                      │
│  POST https://api.siliconflow.cn/v1/audio/transcriptions             │
│  模型: FunAudioLLM/SenseVoiceSmall（免费）                            │
│  输入: WAV bytes → 输出: 文本字符串                                   │
│                                                                      │
│  发给前端: {"type":"asr_result", "text":"今天天气怎么样"}              │
│                                                                      │
│  如果识别为空 → {"type":"error", "message":"未检测到语音内容"}         │
│  → 跳过后续阶段，前端移除空气泡                                       │
└──────────────────────────────────────────────────────────────────────┘
                                    │
                                    ▼
┌──────────────────────────────────────────────────────────────────────┐
│  阶段 4：LangGraph 智能体（后端 agent/graph.py）                      │
│                                                                      │
│  Plan-Execute-Replan 循环架构（详见第三章）                           │
│                                                                      │
│  plan_step ──→ 拆解用户问题为有序子任务列表                           │
│       │                                                               │
│       ▼                                                               │
│  execute_step ──→ ReAct Agent 执行计划第 1 步（调用工具）             │
│       │          ┌─ get_current_datetime() 获取当前日期                │
│       │          └─ tavily_search_tool() Web 搜索                     │
│       ▼                                                               │
│  replan_step ──→ 评估结果，决定继续或完成                             │
│       │                                                               │
│       ├── 继续 → 更新计划，回到 execute_step                          │
│       └── 完成 → 生成最终答案                                         │
│                                                                      │
│  状态实时推送前端:                                                     │
│  - "正在分析问题并制定计划…"（plan）                                    │
│  - "正在执行: 搜索xxx"（search）                                       │
│  - "正在评估执行结果…"（replan）                                       │
└──────────────────────────────────────────────────────────────────────┘
                                    │
                                    ▼
┌──────────────────────────────────────────────────────────────────────┐
│  阶段 5：文本回复（后端 → 前端）                                      │
│                                                                      │
│  发给前端:                                                            │
│  {"type":"bot_text", "text":"今天北京的天气晴朗，气温25°C..."}        │
│  {"type":"thinking", "status":"stop"}                                │
│                                                                      │
│  前端 ChatPanel 在对话面板追加机器人文字气泡                           │
└──────────────────────────────────────────────────────────────────────┘
                                    │
                                    ▼
┌──────────────────────────────────────────────────────────────────────┐
│  阶段 6：语音合成 TTS（后端 services/tts.py）                         │
│                                                                      │
│  POST https://api.siliconflow.cn/v1/audio/speech                     │
│  模型: FunAudioLLM/CosyVoice2-0.5B                                   │
│  输入: 机器人回复文本 → 输出: MP3 bytes                               │
│                                                                      │
│  Base64 编码后发给前端:                                                │
│  {"type":"bot_audio", "data":"<base64_mp3>"}                         │
└──────────────────────────────────────────────────────────────────────┘
                                    │
                                    ▼
┌──────────────────────────────────────────────────────────────────────┐
│  阶段 7：语音播放（前端 useWebSocket.js）                             │
│                                                                      │
│  Base64 解码 → Blob → Audio.play()                                   │
│  播放后 0.5s 冷却，避免扬声器回声触发麦克风                            │
│  状态恢复为 idle，用户可进行下一轮对话                                 │
└──────────────────────────────────────────────────────────────────────┘
```

### 对话持久化

每次对话完成后：
1. **Checkpointer** 自动保存完整 state（`messages` + `plan` + `past_steps`）到 PostgreSQL
2. **SessionMeta** 更新 `message_count` + `last_active_at`
3. 下次打开同一会话 → 从 checkpointer 恢复全部上下文

## 三、LangGraph 智能体架构

### 3.1 图结构

```
                    ┌──────────┐
                    │  START   │
                    └────┬─────┘
                         │
                    ┌────▼─────┐
                    │ planner  │  ← plan_step()
                    │ (V4 Pro) │     ChatPromptTemplate + 结构化输出 Plan
                    └────┬─────┘
                         │
                    ┌────▼─────┐
                    │  agent   │  ← execute_step()
                    │(V4 Flash)│     create_agent + ReAct 工具调用
                    └────┬─────┘
                         │
                    ┌────▼─────┐
                    │  replan  │  ← replan_step()
                    │ (V4 Pro) │     ChatPromptTemplate + 结构化输出 Act
                    └────┬─────┘
                         │
              ┌──────────┼──────────┐
              │ response             │ next_steps
              ▼                     ▼
         ┌──────┐             ┌──────────┐
         │ END  │             │  agent   │  (循环)
         └──────┘             └──────────┘
```

### 3.2 State 定义（`tools.py`）

```python
class AgentState(TypedDict):
    messages: Annotated[list, add_messages]   # 对话历史（短期记忆）
    input: str                                 # 当前用户问题
    plan: list[str]                            # 有序执行步骤
    past_steps: Annotated[list[tuple], operator.add]  # 累积 (步骤, 结果)
    response: str                              # 最终答案
```

- `messages`：`add_messages` reducer 确保多轮对话不丢失
- `past_steps`：`operator.add` reducer 累积每步执行结果，不覆盖
- `plan` / `response`：普通覆盖语义

### 3.3 节点详解

#### plan_step — 问题拆解

- 模型：**DeepSeek V4 Pro**（需要强推理）
- 输出：Pydantic `Plan` 模型，结构化 JSON `{"steps": [...]}`
- 时间规则：如果问题含"最新"/"latest"等词，第一步必须是"获取当前日期"
- 注入对话历史作为上下文

#### execute_step — 工具执行

- 模型：**DeepSeek V4 Flash**（快，擅长工具调用）
- 架构：`create_agent(model, tools, prompt)` 创建 ReAct Agent
- 只执行计划的**第一步**（`plan[0]`）
- 可利用的工具：
  - `get_current_datetime()` → 获取当前 UTC 时间
  - `tavily_search_tool(query)` → Web 搜索（advanced 深度，最多 3 条结果）
- Agent 可自主决定调用哪些工具、调用几次
- 返回结果通过 `operator.add` 追加到 `past_steps`

#### replan_step — 评估与决策

- 模型：**DeepSeek V4 Pro**
- 输出：Pydantic `Act` 模型
  - `{"response": "..."}` → 任务完成，给出最终答案
  - `{"next_steps": [...]}` → 继续执行，返回优化后的剩余步骤
- 如果所有步骤完成 → 汇总生成口语化最终答案
- 决策条件：
  - 还有未完成步骤 → `next_steps`
  - 所有步骤完成 + 信息充分 → `response`
  - 绝不使用模型内部知识，只基于搜索结果

### 3.4 两层模型策略

| 模型 | 节点 | 原因 |
|------|------|------|
| `deepseek-v4-pro` | planner, replanner | 需要复杂推理、结构化输出、上下文理解 |
| `deepseek-v4-flash` | executor (ReAct Agent) | 只需工具调用，flash 快 3-5 倍，降低延迟 |

### 3.5 取消机制

```
用户点"终止" → 前端发送 {"type":"cancel"}
  → ws.py 调用 set_cancel_event(session_id)
  → 每个节点开头 _check_cancelled() 检查 asyncio.Event
  → 若已设置 → 抛出 CancelledError
  → ws.py 捕获 → 发送 cancelled + thinking stop
  → clear_cancel_event() 重置标志
```

每次新 audio 消息到来时也会 `clear_cancel_event()`，确保前一次取消不影响后续。

## 四、记忆系统

### 4.1 短期记忆（对话上下文）

| 机制 | 实现 |
|------|------|
| Checkpointer | `AsyncPostgresSaver`，自动保存每个节点后的完整 state |
| 对话历史 | `messages` 字段（`add_messages` reducer），逐轮累积 |
| 注入方式 | `plan_step` 和 `replan_step` 通过 `_get_conversation_history()` 提取最近 10 轮对话，注入 LLM prompt |
| TTL | 14 天，APScheduler 每天凌晨 3:00 清理 |

### 4.2 长期记忆（历史会话列表）

| 机制 | 实现 |
|------|------|
| 会话元数据 | Tortoise `SessionMeta` 模型：`thread_id`, `title`, `created_at`, `last_active_at`, `message_count` |
| 创建时机 | 首次收到语音消息时自动创建（`_upsert_session`） |
| 前端展示 | Sidebar 每 5 秒轮询 `GET /api/sessions` |
| 切换会话 | 点击历史会话 → `GET /api/sessions/{id}/messages` → 从 checkpointer 恢复消息 → 渲染到面板 |

## 五、前端架构

### 5.1 组件树

```
App (WebSocket Pool 管理)
├── Sidebar
│   ├── NewChatButton        → 新建会话
│   └── SessionList           → 历史会话列表（5s 轮询）
│       └── SessionItem × N   → 点击切换
├── ChatPanel
│   ├── StatusIndicator       → 就绪 / 聆听中 / 处理中
│   ├── ErrorBanner           → 红色错误提示（5s 自动消失）
│   ├── MessageList           → 滚动消息列表
│   │   ├── Message (user)    → 蓝色气泡，右侧
│   │   ├── Message (assistant) → 深色气泡，左侧
│   │   └── Thinking          → 动态状态文字 + 终止按钮
│   └── RecordBar             → 🎤 开始 / ⏹ 结束 按钮
└── SettingsModal             → 模型/API Key 配置弹窗
```

### 5.2 状态管理

```javascript
state = {
  currentSessionId: "abc-123",
  sessions: [...],           // 历史会话列表
  sessionStates: {           // 每个会话独立状态
    "abc-123": { messages: [...], status: "idle", statusText: "" },
    "def-456": { messages: [...], status: "processing", statusText: "正在搜索..." },
  },
  settingsModalOpen: false,
  error: null,
};
```

关键设计：`sessionStates` 按 `sessionId` 隔离，切换会话不影响其他会话的 WebSocket 消息接收和状态更新。

### 5.3 WebSocket 连接池

```
App.jsx 管理全局 wsPoolRef = {
  "abc-123": WebSocket (连接中),
  "def-456": WebSocket (连接中),
}

切换会话 ≠ 关闭旧连接 → 后台继续处理
删除会话 → close() → 释放连接
```

## 六、数据库架构（PostgreSQL）

### 6.1 表结构

| 表 | 管理方 | 用途 |
|----|--------|------|
| `settings` | Tortoise-ORM | 模型配置单行（model_name, base_url, api_key, tavily_key, proxy_url） |
| `session_meta` | Tortoise-ORM | 会话列表元数据（thread_id PK, title, created_at, last_active_at, message_count） |
| `checkpoints` | AsyncPostgresSaver | LangGraph checkpoint 快照 |
| `checkpoint_writes` | AsyncPostgresSaver | Pending writes |
| `checkpoint_blobs` | AsyncPostgresSaver | Channel data blobs |

### 6.2 连接信息

`.env` 文件管理：
```bash
PG_HOST=192.168.0.100
PG_PORT=5432
PG_USER=postgres
PG_PASSWORD=***
PG_DB=voice_chat
```

`config.py` 加载 `.env` → `DATABASE_URL` → Tortoise + AsyncPostgresSaver 共用。

## 十、启动方式

```bash
# 1. 启动 PostgreSQL
docker-compose up -d postgres

# 2. 启动后端 + 前端（一条命令）
cd backend
python manage.py start

# 3. 查看状态
python manage.py status

# 4. 停止
python manage.py stop
```

浏览器打开 `http://localhost:5173`，点 🎤 或按空格键开始说话。
