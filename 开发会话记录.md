# 开发会话记录

## 项目概述

纯语音聊天机器人，中英文，火山引擎 ASR/TTS，DeepSeek V4 Pro，LangGraph 1.2 智能体，FastAPI + React + PostgreSQL。

## 第一阶段：架构设计确认

### 核心技术选型（已确认）

| 决策项 | 结论 |
|--------|------|
| ASR | 火山引擎 - 豆包录音文件识别模型 2.0（0.8 元/小时） |
| TTS | 火山引擎 - 豆包语音合成模型 2.0（3 元/万字符） |
| 前端 | React 18 + Vite，无 UI 框架 |
| 后端 | FastAPI + Tortoise-ORM |
| LLM | DeepSeek V4 Pro（模型名/base_url/API Key 可配置） |
| Python | 3.12 |
| 搜索 | Tavily Search（`langchain_tavily`），无结果→回答"未找到"，不降级到 LLM 直接回答 |
| 终止机制 | asyncio.Event + checkpoint 回滚 |
| TTL 清理 | APScheduler 每天凌晨 3 点，查询时也过滤 |
| 代理 | settings 表预留 proxy_url 字段，本地阶段先不配置 |
| 部署 | 本地开发 → 未来 Docker 上云 |
| 并发 | 支持多标签页/多 session，单用户无登录 |

### 音频处理

- VAD：浏览器 `AudioContext` + `AnalyserNode`，RMS 能量检测，连续 3 秒低于阈值→结束
- 格式：16kHz, 16-bit, Mono WAV → Base64 传输
- 策略：整句说完后整体处理（非流式），LLM 全量生成完后一次性 TTS

### WebSocket 协议（9 种消息类型）

客户端→服务端: `audio`, `cancel`, `ping`
服务端→客户端: `asr_result`, `thinking`(start/stop), `bot_text`, `bot_audio`, `cancelled`, `error`, `pong`

### LangGraph 工作流

```
START → plan_node → search_node → synthesize_node → END
```

- **plan_node**: LLM 拆解用户问题为有序子问题（JSON），含依赖关系
- **search_node**: 按 DAG 关系并行/串行搜索，Tavily 无结果标记 no_results
- **synthesize_node**: LLM 汇总搜索结果生成口语化答案（含 System Prompt）

### 数据库表

| 表 | 管理方式 | 用途 |
|----|---------|------|
| settings | Tortoise-ORM | 模型配置（单行） |
| session_meta | Tortoise-ORM | 会话列表元数据 |
| checkpoints | AsyncPostgresSaver 自动管理 | LangGraph checkpoint |
| checkpoint_writes | AsyncPostgresSaver 自动管理 | pending writes |
| checkpoint_blobs | AsyncPostgresSaver 自动管理 | channel data |

## 第三阶段：LangGraph Checkpointer + Store 实现

### 关键决策

1. **Checkpointer 单例化**：`db/langgraph.py` 管理 `AsyncPostgresSaver`，app 启动时初始化一次，所有请求共享
2. **Session 元数据**：用 Tortoise `SessionMeta` 模型而非 LangGraph Store，查询更可靠
3. **消息类型标准化**：`AgentState` 使用 TypedDict + `add_messages` reducer，消息用 `HumanMessage`/`AIMessage`
4. **Graph 编译缓存**：按 checkpointer 实例 id 缓存编译后的 graph，避免重复编译

### 已修复的代码缺陷

- `graph.py`：`dict[int, any]` → `dict[int, Any]`
- `nodes.py`：删除未使用的 `_is_user_message()` 函数
- `ws.py`：删除未使用的 `is_first` 参数和 `first_turn` 变量
- `tools.py`：删除未使用的 `Any` import

### Tavily 改用 langchain_tavily

- `tavily.TavilyClient` → `langchain_tavily.TavilySearch`
- `requirements.txt`：`tavily-python` → `langchain-tavily`

### 验证方式

`backend/verify_checkpointer.py` — 6 步验证脚本，需先 `docker-compose up postgres`
